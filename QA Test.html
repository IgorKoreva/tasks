Результаты

Case 1

Your Answer: frontend - Не работает кнопка "Зарегистрироваться". При нажатии ничего не происходит - в devtools, вкладки console и network пусты.

Correct Answer: Баг фронтенда: форма регистрации не отправляет запрос на сервер при попытке зарегистрироваться. Это можно проверить, открыв вкладку «Netwokr» в инструментах разработчика браузера.

Case 2

Your Answer: frontend - Ошибка 403 Forbidden (Запрещено), т.к. по документации API мы должны передавать "4е поле" - "inviteCode", но его на фронте нет.

Correct Answer: Баг фронтенда: в форме регистрации отсутствует обязательное поле для ввода инвайт-кода, который в данном случае необходим для регистрации. Бэкенд правильно возвращает ошибку 403 Forbidden, поскольку инвайт-код отсутствует. Данный код можно найти в запросах на получение информации о пользователях.

Case 3

Your Answer: frontend - 500 ISE. Передается неверный JSON - отсутствуют кавычки у ключей (network/payload/). Backend не обрабатывает не валидный JSON - ожидаемая ошибка 400 Bad Requests.

Correct Answer: Баг бэкенда: сервер возвращает ошибку 500 Internal Server Error. Это общее сообщение об ошибке, которое указывает на то, что на сервере возникло непредвиденное условие.

Case 4

Your Answer: frontend - 500 ISE. Неверный эндпойнт: В документации указан /register, фронт стучится на /register-bad

Correct Answer: Баг фронтенда: фронтенд отправляет запрос на регистрацию на неверный URL-адрес (http://localhost:3000/register-bad). Правильный URL-адрес: http://localhost:3000/register.

Case 5

Your Answer: frontend - 500 ISE. В request фронтом передается неверная модель. Сервер ждет json с "name", "email", "password", а мы передаем (network/payloads/) "e", "n", "p"

Correct Answer: Баг фронтенда: фронтенд отправляет запрос с искаженным телом. Ключи для имени, электронной почты и пароля сокращены до n, e и p, которые сервер не понимает.

Case 6

Your Answer: frontend - 500 ISE. Используется неверный метод - PUT. По документации API мы должны использовать POST.

Correct Answer: Баг фронтенда: фронтенд отправляет запрос PUT вместо запроса POST. Сервер не настроен для обработки запросов PUT для конечной точки /register.
